## On the project

sluice is maturing well. The pure-core / shell split remains the strongest part of the
architecture — every new feature (windowed phantom, gate_closed_reason, status projection)
slotted into the right layer without strain. The import boundary test catches violations
and has survived every addition. The AGENTS.md hard rules (fail-safe, streaming sacred,
cache-transparency) are load-bearing in the design, not just documentation — each one
shaped concrete implementation decisions this session.

The singleton guard is the most architecturally significant addition. It transforms the
single-instance invariant from "a deployment convention with a CRITICAL comment" into
"a mechanism that fails loudly." The KubeLeaseGuard using raw httpx REST (no heavy
kubernetes client) is a good call — the k8s API is just REST, and we already depend on httpx.

The status web UI is functional but bare-bones. The gate ladder renders, the Prometheus
metrics scrape, and the admin token gates the data endpoints. But it's missing SSE streaming
(WI-003 of Plan 002) and the patina design system (WI-007). The 1-second polling is
acceptable for now but will feel laggy under fast-changing state.

## On the work done

**What went well:**
- The windowed phantom estimator was the highest-value fix. The old `max(0, observed -
  local_in_flight)` comparing a lagged reading against current permits would over-throttle
  under normal churn. The new `max(0, min_i(observed_i - local_i))` over K samples drops
  transient release-lag while sustaining real phantoms. The pure function is clean and
  well-tested.
- The adversarial review caught real issues. The most important was `release()` deleting
  the lease unconditionally — if pod A's lease expires, pod B takes over, and A shuts down,
  A would delete B's lease. Also `record_success()` firing on response headers instead of
  after stream completion — a half-open breaker probe that disconnects mid-stream would
  count as success. Both are the kind of subtle bug that would only surface in production
  under failure conditions.
- The KubeLeaseGuard tests against a FakeLeaseAPI are comprehensive: acquire-when-free,
  refuse-when-held, reacquire-after-expiry, renew-success, renew-failure-flips-held,
  held-by-another-flips-held, release-deletes, api-error-fails-safe.

**What was awkward:**
- The httpx MockTransport doesn't support async streaming. The `_AsyncMockTransport` in
  test_proxy.py works around this by re-wrapping content as async generators, but it's
  load-bearing test infrastructure. Changing it to use `request.aread()` (so the body is
  available for inspection) was needed for the cache-transparency byte-identity test.
- The dashboard HTML is a large string constant in proxy.py. It works but it's not ideal —
  a separate template file would be cleaner. The inline approach was chosen to avoid adding
  a template dependency or build step.
- The admin token auth design has a tension: the dashboard page (`/`) needs to be accessible
  without auth (so the JS can load), but `/status.json` requires auth. The JS can't send the
  bearer token. Currently the dashboard is served without auth (it's just a static page),
  and the data endpoint is gated. This means the dashboard can't fetch data when admin token
  is set — the JS needs to include the token. This is a known gap.

**What I'd want a second pair of eyes on:**
- The KubeLeaseGuard's `is_held()` doesn't check local lease expiry. If the event loop is
  blocked longer than lease_duration, `is_held()` returns True but the lease has expired
  server-side. This is the classic split-brain issue. Filed as WI-003.
- The phantom window's interaction with the staleness path. When the reading is stale,
  phantom_estimate is not applied (correct — don't trust the number). But the flat staleness
  penalty (`target - stale_penalty`) doesn't account for priority_low. If priority_low=True
  and the reading is stale, the gate could be looser than when fresh. The monotonicity test
  passes (stale ≤ baseline), but the comparison stale-vs-fresh-with-phantoms isn't tested.
- The `reconcile.run()` exception handler now closes the gate on unexpected exceptions.
  But if `gate.resize(0)` itself fails, the log.critical fires and the loop continues.
  That's the best we can do — if the event loop is broken, sluice can't protect itself.

## Gaps to flag

- **`singleton.py:265`**: `is_held()` returns `_held` without checking time since last renew.
  If the event loop is blocked > lease_duration, another pod acquires but this pod keeps
  admitting. Filed as WI-003.
- **`proxy.py:144-152`**: Non-leader never re-attempts acquire after startup failure. The
  pod serves 503s forever until restart. Filed as WI-004.
- **`reconcile.py:270-291`**: `retry_after_seconds()` uses stale `resets_at` when
  `cached.ok=False`. Could tell clients to wait hours for a box that already lifted.
  Filed as WI-006.
- **`proxy.py:214`**: `body_queue` has no `maxsize`. A slowloris-style attack with a permit
  can exhaust memory. Filed as WI-007.
- **`proxy.py` dashboard HTML**: The JS `fetch('/status.json')` doesn't include the admin
  token. When `--admin-token` is set, the dashboard loads but can't fetch data. Needs either
  a token-in-URL approach or a cookie-based session. Not filed yet — it's a UX issue, not a
  correctness bug, and the dashboard is secondary to the CLI `sluice status` command.
- **`control.py:150-158`**: The staleness path (`base = min(base, target - stale_penalty)`)
  doesn't account for `priority_low` in the cap. If `priority_low=True` and the reading is
  stale with a large phantom_estimate, the stale path can give more permits than the fresh
  path. This is a design trade-off (stale = don't trust phantoms, apply flat penalty), but
  the interaction is subtle and not tested.
- **`test_proxy.py` `_AsyncMockTransport`**: Changed from `async for _ in request.stream` to
  `await request.aread()`. This makes the body available for inspection in test handlers.
  If httpx changes this behavior in a future version, tests may break silently.
- **`singleton.py:92-104`**: `_now_rfc3339()` hardcodes `.000000Z` (microsecond truncation).
  The k8s API truncates to seconds anyway, so this is harmless but worth noting.
- **`deploy/k8s/deployment.yaml`**: Still uses `image: ghcr.io/hraedon/sluice:latest` with
  `imagePullPolicy: IfNotPresent`. Updates require deleting the pod or changing the tag.
  Consider pinning to a version tag once the first release is cut.
